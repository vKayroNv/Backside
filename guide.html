Backside Engine

Руководство по использованию пакетов ru.kayron.cargo (DI-контейнер) и ru.kayron.dew (игровой движок на OpenGL ES 3.0 для Android).

1. Пакет ru.kayron.cargo — Dependency Injection

Обзор

Cargo — это лёгкий IoC-контейнер с поддержкой трёх времени жизни (Lifetime): SINGLETON, SCOPED, TRANSIENT. Контейнер умеет автоматически разрешать зависимости через primary-конструктор, поддерживает Named-квалификаторы, ленивую загрузку (Lazy), провайдеры (Provider), вложенные скоупы и отложенную инициализацию.

object Cargo

object Cargo {
    val root = CargoContainer()
}

Описание: Точка входа. Содержит единственный корневой контейнер. Все игровые объекты используют Cargo.root как глобальный DI-контейнер.

class CargoContainer

class CargoContainer(private val parent: CargoContainer? = null)
    : AutoCloseable

Методы регистрации

addSingleton — регистрирует singleton-зависимость:

fun <T: Any> addSingleton(
    type: KClass<T>,
    qualifier: String? = null,
    factory: CargoContainer.() -> T
)

Reified-версия: inline fun <reified T : Any> addSingleton(qualifier: String? = null, noinline factory: CargoContainer.() -> T)

Перегрузка с готовым экземпляром: fun <T : Any> addSingleton(type: KClass<T>, instance: T)

addScoped — регистрирует scoped-зависимость (живёт в пределах дочернего контейнера):

fun <T: Any> addScoped(
    type: KClass<T>,
    qualifier: String? = null,
    factory: CargoContainer.() -> T
)

addTransient — регистрирует transient-зависимость (новый экземпляр при каждом запросе):

fun <T: Any> addTransient(
    type: KClass<T>,
    qualifier: String? = null,
    factory: CargoContainer.() -> T
)

bindSingleton / bindScoped / bindTransient — связывание абстракции с реализацией без фабрики (авто-создание через конструктор):

fun <TAbstraction: Any, TImplementation: TAbstraction>
    bindSingleton(
    abstraction: KClass<TAbstraction>,
    implementation: KClass<TImplementation>,
    qualifier: String? = null
)

addConfig — регистрирует конфигурацию как singleton:

fun <T: Configuration> addConfig(
    type: KClass<T>,
    config: T
)

Методы получения

get — получает зарегистрированную зависимость:

fun <T: Any> get(
    type: KClass<T>,
    qualifier: String? = null
): T

// reified inline-версия:
inline fun <reified T: Any> get(
    qualifier: String? = null
): T

create — создаёт экземпляр вне регистрации, передавая внешние аргументы в конструктор:

fun <T: Any> create(
    type: KClass<T>,
    vararg arguments: Any
): T

Управление контейнером

load(module: CargoModule) — загружает модуль с набором регистраций.
freeze() — замораживает контейнер, запрещая дальнейшие регистрации.
build() — валидирует граф зависимостей, создаёт eager-синглтоны и замораживает контейнер.
scope() — создаёт дочерний контейнер с родительской цепочкой.
dispose() — освобождает все scoped и singleton-экземпляры (только для root-контейнера).

Примеры использования Cargo

Базовая регистрация и получение

// регистрация с фабрикой
Cargo.root.addSingleton {
    MyService()
}

// получение
val service = Cargo.root.get<MyService>()

// binding абстракции к реализации
Cargo.root.bindSingleton<IService, MyService>()

// получение через интерфейс
val service: IService = Cargo.root.get()

Named-квалификаторы

// регистрация с квалификатором
Cargo.root.addSingleton<Database>("users") {
    Database("users.db")
}
Cargo.root.addSingleton<Database>("logs") {
    Database("logs.db")
}

// получение с квалификатором
val usersDb = Cargo.root.get<Database>("users")

// использование аннотации @Named в конструкторе
class ReportService(
    @Named("users") private val db: Database
)

Lazy и Provider

class Consumer(
    private val lazyService: Lazy<HeavyService>,  // создаётся при первом обращении
    private val provider: Provider<TransientService>  // каждый get() даёт новый экземпляр
)

Модули

val databaseModule = module {
    singleton<Database> { Database("app.db") }
    transient<Repository> { Repository(get()) }
}

Cargo.root.load(databaseModule)
Cargo.root.build()

Скоупы

val childScope = Cargo.root.scope()
childScope.addScoped { Session() }

val session = childScope.get<Session>()

// scoped-объекты уничтожаются при dispose скоупа
childScope.dispose()

7. Математика

Vector2 / Vector3 / Vector4

Data-классы с компонентами x, y (z, w). Поддерживают:

  • Арифметические операторы (+, -, *, /, unary -)
  • length(), lengthSquared(), normalize()
  • Статические методы: add, subtract, multiply, divide, negate, dot, cross, distance, lerp, normalize, reflect, min, max, clamp, transform
  • toArray()
  • Vector2 дополнительно: angle(), transformNormal()
  • Vector3: дополнительные константы Up, Down, Right, Left, Forward, Backward

Matrix

Матрица 4×4 (m11...m44). Реализованы:

  • Статические методы: Identity, createTranslation, createScale, createRotationX/Y/Z, createOrthographic, createOrthographicOffCenter, createPerspectiveFieldOfView, createLookAt, createFromQuaternion, createWorld, transpose, multiply, invert
  • instance-методы: determinant(), toArray(), translation(), scale(), forward/backward/up/down/right/left(), decomposeScale()
  • Операторы: * (Matrix × Matrix), * (Matrix × Vector3)

Color / Rectangle / Quaternion / Point

Color

Формат: packedValue (UInt, RRGGBBAA). Конструкторы: (r, g, b, a: Int), (r, g, b, a: Float), (Vector3), (Vector4). Свойства r, g, b, a (Int 0-255), rf, gf, bf, af (Float 0-1). 80+ именованных констант. Операторы: *, +.

Rectangle

Свойства: x, y, width, height, left, right, top, bottom, center, location, isZero, isEmpty. Методы: contains, inflate, intersects, intersect, union, offset, toFloatArray.

Quaternion

Компоненты x, y, z, w. Константа Identity. Статические методы: createFromAxisAngle, createFromYawPitchRoll, createFromRotationMatrix, conjugate, inverse, dot, lerp, slerp, normalize. Instance: toMatrix().

Point

x, y (Int). Константы Zero, One. Операторы: +, -, *, /, unary -. Метод: toVector2().

MathHelper

Константы: E, Log10E, Log2E, Pi, PiOver2, PiOver4, TwoPi. Методы: clamp, distance, lerp, smoothStep, hermite, catmullRom, toRadians, toDegrees, wrapAngle, max, min, sqrt.

        sceneManager.initialize()
